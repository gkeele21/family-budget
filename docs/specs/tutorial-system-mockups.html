What is "Ready to Assign"?

🤓
Think of it like cash sitting on your kitchen table. It's real money you have, but you haven't decided what it's for yet.

Ready to Assign shows how much money you have that hasn't been assigned to a category yet.

It's calculated as:

+ All your income
− Everything you've assigned
= Ready to Assign

Your goal is to get this to $0.00 — that means every dollar has a job. It doesn't mean you're broke!

⚠️ If it's negative...
You've assigned more money than you actually have. Go to your categories and reduce some amounts until Ready to Assign is $0 or more.

7. Future Considerations — Not Yet Built

Integration points and enhancements needed to fully wire the tutorial system into the existing app. These are documented here for future implementation.

🎯

Add data-tutorial Attributes to Budget Page

The tutorial overlay highlights elements using CSS selectors like [data-tutorial="ready-to-assign"]. These attributes need to be added to the actual budget page components so the highlight targeting works.

data-tutorial="ready-to-assign" → AvailableToBudget component
data-tutorial="category-list" → Category groups wrapper
data-tutorial="category-rent" → First Bills category row (Rent)
data-tutorial="category-groceries" → Groceries category row
data-tutorial="fab-button" → FAB / add transaction button
🔌

Integrate TutorialOverlay into Budget Page Layout

The TutorialOverlay component needs to be rendered on the Budget page when an active tutorial is detected. Use the useTutorial() composable to check isActive and pass the current step/track to the overlay.

// In Budget/Index.vue or AppLayout.vue:
const { isActive, track, currentStep, steps, advance, skip } = useTutorial();
<TutorialOverlay
  :active="isActive"
  :track="track"
  :step="currentStep?.id"
  :steps="steps"
  @primary="advance"
  @secondary="skip"
/>
⚙️

Add "Help & Tutorials" Link in Settings

Add a row in the Settings page that links to /tutorial (the Tutorial Hub). This lets existing users access tutorials and Quick Tips at any time, not just on first sign-up. Consider grouping it near existing support/about links if any exist.

🚪

Redirect New Users to Tutorial Hub After Sign-Up

After a new user completes registration, redirect them to /tutorial instead of straight to the budget page or onboarding. This gives them the choice between learning first (sandbox) or diving into setup. Could be done in the RegisteredUserController or post-registration middleware.

❓

Tutorial Budget Cleanup

When the Learn tutorial completes, offer to delete the tutorial budget (or auto-delete it). The completion step's "Set Up My Budget →" action should switch the user away from the tutorial budget and optionally clean it up. Also consider preventing tutorial budgets from appearing in the budget switcher after completion, or marking them with a visual indicator.

📦

Empty State Education

When sections are empty (no transactions, no budgeted categories, no accounts), show helpful guidance instead of blank screens. Examples:

"You haven't assigned any money this month — tap a category to get started"
"No transactions yet — record your first purchase with the + button"
"No recurring bills set up — add regular payments so you never miss one"
🔔

Gentle Nudges & Reminders

As mentioned in the Setup completion screen, the mascot could remind users if they haven't logged transactions in a while. Could be implemented as a banner on the budget page or a notification. Also consider nudges for: unassigned money sitting in Ready to Assign, overspent categories not yet resolved, new month starting without assignments.

🔄

Replay & Resume Tutorials

Allow users to replay tutorials from the Tutorial Hub even after completing them (the "Completed" badge already shows, but the card should still be tappable). If a user exits mid-tutorial, save their progress so they can resume where they left off (the tutorial_step field already supports this — just need UI to offer "Resume" vs "Start Over" when returning to the Hub).
